## How to Upgrade

Follow these steps to upgrade Squbix to the latest version. Always back up your data first.

<Steps>

<Step title="Backup Your Data" icon="database">

Export patient records and configurations via the admin panel.

```bash
# Use CLI for full backup
squbix backup --output ./backups --env production
```

</Step>

<Step title="Update Package" icon="package">

<CodeGroup tabs="npm,yarn">

````bash
npm update @squbix/core @squbix/billing
````

````bash
yarn upgrade @squbix/core@latest @squbix/billing@latest
````

</CodeGroup>

Verify the update:

```bash
squbix version
```

</Step>

<Step title="Run Migrations" icon="settings">

Apply database schema changes.

```bash
squbix migrate --env production
```

Test endpoints like `https://api.example.com/v2/patients`.

</Step>

<Step title="Restart Services" icon="refresh-cw">

```bash
systemctl restart squbix
# Or use Docker
docker-compose up -d
```

</Step>

</Steps>

<Expandable title="Migration Notes for v2.0.0" default-open="false">

If upgrading from v1.x, update your auth calls:

<CodeGroup tabs="JavaScript,Python">

````javascript
// Old
const response = await fetch('https://api.example.com/patients', {
  headers: { 'X-API-Key': 'YOUR_API_KEY' }
});

// New
const token = await getJWTToken();
const response = await fetch('https://api.example.com/v2/patients', {
  headers: { 'Authorization': `Bearer ${token}` }
});
````

````python
# Old
response = requests.get('https://api.example.com/patients', headers={'X-API-Key': 'YOUR_API_KEY'})

# New
token = get_jwt_token()
response = requests.get('https://api.example.com/v2/patients', headers={'Authorization': f'Bearer {token}'})
````

</CodeGroup>

</Expandable>
